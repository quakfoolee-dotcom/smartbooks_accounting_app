# Issue 84 Playwright Diagnostics Snapshot

Date: 2026-07-01

## Objective

Capture CI failure artifacts for Playwright diagnostics so intermittent browser failures include traces, screenshots, videos, reports, and concise summaries.

## Before And After Fix Report

| Area | Before | After |
|---|---|---|
| Playwright reporters | Local and CI runs used the `list` reporter only. | CI now also writes HTML and JSON reports while local runs stay lightweight. |
| Failure artifact bundle | CI uploaded `test-results/` and `playwright-report/`, but `playwright-report/` was not generated by the config. | Failed CI shards upload `playwright-diagnostics/`, `test-results/`, and `playwright-report/`. |
| Failure summary | Maintainers had to infer failures from job logs and raw artifact files. | `scripts/write-playwright-diagnostics.js` writes a shard summary and manifest from Playwright JSON output. |
| Artifact names | Functional shard artifacts were named `playwright-failure-artifacts-*`. | Artifacts are named `playwright-diagnostics-*` by shard, with `pages-smoke-diagnostics` for live smoke failures. |
| Retention | Browser failure artifacts were retained for 14 days. | Playwright diagnostics retain for 7 days to reduce storage noise. |
| Documentation | CI docs mentioned downloading artifacts but did not explain the diagnostic workflow. | `docs/playwright-ci-diagnostics.md` explains when artifacts appear, what they contain, and how to inspect reports/traces. |

## Diagnostic Workflow

| Step | Action |
|---|---|
| Identify | Open the failed GitHub Actions shard job rather than the aggregate `CI Result`. |
| Download | Download `playwright-diagnostics-<shard>` or `pages-smoke-diagnostics`. |
| Summarize | Read `playwright-diagnostics/*-summary.md` first. |
| Inspect | Use `npx playwright show-report playwright-report` and `npx playwright show-trace path\to\trace.zip`. |
| Fix | Reproduce with the matching shard command, fix the defect, and rerun validation. |
